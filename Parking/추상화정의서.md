# [주제 선정 및 객체 추상화 정의서]

## 1. 주제 및 기획 배경
* **주제**: 파일 입출력 기반의 주차장 관리 프로그램
* **기획 배경**: 주차장의 요금 정산과 남은 자리를 효율적으로 관리하고, 차량 데이터를 텍스트 파일로 영구 저장하여 프로그램 재실행 시에도 연속적인 데이터 관리가 가능하도록 하기 위함.

---

## 2. 시스템 설계 구조 (MVC + File Utility)

| 구분 | 클래스/객체명 | 역할 및 책임 | 주요 속성 (Attributes) | 주요 기능 (Methods) |
| :---: | :--- | :--- | :--- | :--- |
| **Model** | `com.kh.model.vo.Car` | 주차 차량의 정보(차량번호, 차주명, 연락처)와 정산에 필요한 입차 시간을 표현하는 데이터 모델 | • `carNumber` (차량번호)<br>• `ownerName` (차주명)<br>• `phone` (연락처)<br>• `entryTime` (입차시간) | • 각 필드의 getter / setter<br>• `toString()` : 차량 정보 문자열 반환 |
| **Controller** | `com.kh.controller.ParkingController` | 전체 차량 CRUD 제어, 출차 시 시간 및 요금 정산 비즈니스 로직 처리, 파일 입출력 기능 중재 | • `cars` (Car[] 배열)<br>• `count` (현재 주차 대수)<br>• `fm` (FileManager 객체) | • `addCar(car)` : 차량 등록 (배열 검사 및 중복 체크 포함)<br>• `searchCar(carNumber)` : 차량 검색 (동일 번호 확인)<br>• `updateCar(...)` : 차주명 및 연락처 수정<br>• `processReceipt(carNumber)` : 이용시간/요금 계산 및 출차 확정<br>• `deleteCar(carNumber)` : 배열 데이터 삭제 후 순서 당김 정렬<br>• `getCars()` : 현재 주차된 차량 배열 반환<br>• `saveFile()` / `loadFile()` : 파일 저장 및 로드 위임 |
| **View** | `com.kh.view.ParkingView` | 사용자에게 메뉴 인터페이스를 콘솔로 제공하고, 입력 데이터 접수 및 Controller의 실행 결과를 화면에 출력 | • `sc` (Scanner 객체)<br>• `pc` (ParkingController 객체) | • `mainMenu()` : 주차장 관리 메인 화면 무한 루프 구동<br>• `inputCar()` : 사용자로부터 새 차량 정보(번호, 이름, 폰)를 입력받아 객체 생성<br>• `inputCarNumber()` : 검색/수정/출차용 차량 번호 입력 받기<br>• `printCar(car)` : 단일 차량 상세 정보 양식 출력<br>• `printAllCars(...)` : 전체 주차 현황 리스트 화면 출력<br>• `printMessage(msg)` : 프로그램 알림 및 결과 메시지 출력 |
| **File** | `com.kh.file.FileManager` | 주차 차량 데이터를 외부 텍스트 파일(`parking_data.txt`)에 영구 저장하고, 프로그램 시작 시 복구하는 파일 데이터 영속성 관리 | • `FILE_NAME` (데이터 파일 경로 문자열) | • `save(cars, count)` : 콤마(`,`) 구분자를 기반으로 차량 정보를 파일에 줄 단위로 기록<br>• `load()` : 파일 내용을 읽어 콤마 파싱(Split) 후 `Car` 객체 배열로 복원 및 반환 |
